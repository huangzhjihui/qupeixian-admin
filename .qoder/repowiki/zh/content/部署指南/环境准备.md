## 简介

趣配鲜是一个基于 Node.js 的电商配送平台，需要在服务器上进行完整的环境准备。本指南详细说明了服务器硬件要求、操作系统兼容性、依赖安装步骤以及环境验证方法。

## 服务器硬件要求

根据项目的实际运行需求，建议以下硬件配置：

### 最低配置要求
- **CPU**: 至少 2 核心处理器
- **内存**: 至少 2GB RAM
- **存储空间**: 至少 20GB 可用磁盘空间
- **带宽**: 建议 10Mbps 以上稳定带宽

### 推荐配置要求
- **CPU**: 4 核心或更高
- **内存**: 4GB 或更高
- **存储空间**: 50GB 或更高
- **带宽**: 100Mbps 以上

### 性能考虑因素
- 应用程序需要处理用户请求、图片上传和数据库操作
- MySQL 数据库需要足够的内存进行缓存
- Nginx 需要处理静态文件和反向代理
- PM2 需要管理 Node.js 应用进程

```bash

# 安装 Node.js 18
sudo yum install -y nodejs

# 验证安装
node --version
npm --version
```

#### 方法二：手动下载安装
```bash
# 下载 Node.js 18.x
cd /tmp
wget https://nodejs.org/dist/v18.17.0/node-v18.17.0-linux-x64.tar.xz

# 解压到 /opt
sudo tar -xf node-v18.17.0-linux-x64.tar.xz -C /opt/
sudo mv /opt/node-v18.17.0-linux-x64 /opt/nodejs

# 配置环境变量
echo 'export PATH="/opt/nodejs/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc
```

## Nginx 配置

### Nginx 安装

#### CentOS 7.x 安装
```bash
# CentOS 7 默认仓库中可能没有最新版本的 Nginx
# 可以使用 EPEL 仓库
sudo yum install epel-release -y
sudo yum install nginx -y

# 或者添加 Nginx 官方仓库
sudo tee /etc/yum.repos.d/nginx.repo << EOF
[nginx-stable]
name=nginx stable
baseurl=http://nginx.org/packages/centos/7/x86_64/
gpgcheck=1
enabled=1
gpgkey=https://nginx.org/keys/nginx_signing.key
module_platform_id=platform:el7
EOF

sudo yum install nginx -y
```

#### Ubuntu 20.04+ 安装
```bash
# 使用默认仓库安装
sudo apt update
sudo apt install nginx -y

# 或者添加官方仓库
curl -fsSL https://nginx.org/keys/nginx_signing.key | sudo apt-key add -
echo "deb http://nginx.org/packages/ubuntu/ focal nginx" | sudo tee /etc/apt/sources.list.d/nginx.list
echo "deb-src http://nginx.org/packages/ubuntu/ focal nginx" | sudo tee -a /etc/apt/sources.list.d/nginx.list
sudo apt update
sudo apt install nginx -y
```

### Nginx 基础配置

```bash
# 备份默认配置
sudo cp /etc/nginx/nginx.conf /etc/nginx/nginx.conf.backup

# 创建应用配置文件
sudo tee /etc/nginx/conf.d/qupeixian.conf << EOF
upstream nodejs_backend {
    server 127.0.0.1:3000;
}

server {
    listen 80;
    server_name your_domain.com www.your_domain.com;
    
    location / {
        proxy_pass http://nodejs_backend;
        proxy_http_version 1.1;
        proxy_set_header Upgrade \$http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host \$host;
        proxy_set_header X-Real-IP \$remote_addr;
        proxy_set_header X-Forwarded-Proto \$scheme;
        proxy_set_header X-Forwarded-For \$proxy_add_x_forwarded_for;
        proxy_cache_bypass \$http_upgrade;
    }
    
    location /static/ {
        alias /var/www/qupeixian/static/;
        expires 30d;
    }
    
    error_log /var/log/nginx/qupeixian_error.log;
    access_log /var/log/nginx/qupeixian_access.log;
}
EOF
```

### Nginx 高级配置示例

```bash
# SSL/TLS 配置示例
sudo tee /etc/nginx/conf.d/qupeixian_ssl.conf << EOF
server {
    listen 443 ssl http2;
    server_name your_domain.com www.your_domain.com;
    
    ssl_certificate /path/to/your/certificate.crt;
    ssl_certificate_key /path/to/your/private.key;
    ssl_trusted_certificate /path/to/your/chain.crt;
    
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers ECDHE-RSA-AES256-GCM-SHA512:DHE-RSA-AES256-GCM-SHA512:ECDHE-RSA-AES256-GCM-SHA128:DHE-RSA-AES256-GCM-SHA128;
    ssl_prefer_server_ciphers off;
    
    location / {
        proxy_pass http://nodejs_backend;
        proxy_set_header Host \$host;
        proxy_set_header X-Real-IP \$remote_addr;
        proxy_set_header X-Forwarded-For \$proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto \$scheme;
    }
}
EOF
```

## PM2 进程管理

### PM2 安装

```bash

sudo npm install -g pm2

# 验证安装
pm2 --version

# 查看 PM2 系统信息
pm2 info
```

### PM2 应用配置

```bash
# 在项目根目录创建 PM2 配置文件
sudo tee ecosystem.config.js << EOF
module.exports = {
  apps: [{
    name: 'qupeixian',
    script: './backend/src/app.js',
    instances: 'max',
    exec_mode: 'cluster',
    watch: false,
    max_memory_restart: '1G',
    env: {
      NODE_ENV: 'production',
      PORT: 3000,
      DB_HOST: 'localhost',
      DB_PORT: 3306,
      DB_NAME: 'qupeixian',
      DB_USER: 'qupeixian_user',
      DB_PASSWORD: 'secure_password'
    },
    env_development: {
      NODE_ENV: 'development',
      PORT: 3001
    },
    out_file: '/var/log/qupeixian/out.log',
    error_file: '/var/log/qupeixian/error.log',
    log_date_format: 'YYYY-MM-DD HH:mm:ss'
  }]
}
EOF
```

### PM2 应用启动和管理

## 网络配置要求

### 端口开放要求

| 端口号 | 用途 | 必需性 | 说明 |
|--------|------|--------|------|
| 22 | SSH 远程访问 | 必需 | 服务器管理 |
| 80 | HTTP 服务 | 必需 | Web 服务端口 |
| 443 | HTTPS 加密服务 | 可选但推荐 | 安全传输 |
| 3306 | MySQL 数据库 | 必需 | 数据库连接 |
| 3000 | Node.js 应用 | 必需 | 应用服务端口 |

### 防火墙配置

#### CentOS 7.x (firewalld)
```bash
# 启动防火墙
sudo systemctl start firewalld
sudo systemctl enable firewalld

# 开放必要端口
sudo firewall-cmd --permanent --add-service=http
sudo firewall-cmd --permanent --add-service=https
sudo firewall-cmd --permanent --add-port=3306/tcp
sudo firewall-cmd --permanent --add-port=3000/tcp

# 重新加载配置
sudo firewall-cmd --reload

# 检查开放的端口
sudo firewall-cmd --list-all
```

#### Ubuntu 20.04+ (ufw)
```bash
# 启动防火墙
sudo ufw enable

# 设置默认策略
sudo ufw default deny incoming
sudo ufw default allow outgoing

# 开放必要端口
sudo ufw allow 22/tcp
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw allow 3306/tcp
sudo ufw allow 3000/tcp

# 检查状态
sudo ufw status verbose
```
